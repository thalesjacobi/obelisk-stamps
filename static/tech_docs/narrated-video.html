Narrated Video — Technical Documentation

Source: app.py, templates/article_edit.html Last updated: 2026-03-22

1. Overview

The narrated video system generates slideshow videos from article carousel images with AI-narrated audio. The pipeline runs as follows: GPT script generation (GPT-4.1-mini produces per-slide narration segments) → OpenAI TTS (each segment is synthesised to MP3 audio) → FFmpeg video assembly with a Ken Burns zoom/pan effect for still images, or a cinemagraph loop if a cinemagraph video is available for the slide.

Completed videos can be posted to 8 social platforms: Instagram (Reels), Facebook, YouTube Shorts, X (Twitter), Threads, Pinterest, TikTok, and LinkedIn.

AttributeValue
Max slides10
Output formatsVertical (720×1280), Square (720×720)
Voicesalloy, echo, fable, onyx, nova, shimmer
TTS qualitytts-1 (standard), tts-1-hd (high definition)
Slide animationKen Burns (still images) or Cinemagraph (animated slides)
Social platformsInstagram Reels, YouTube Shorts, Facebook, X, Threads, Pinterest, TikTok, LinkedIn

Multiple generation runs are supported per article. Each run is stored as an object in a JSON array and rendered as an accordion item in the admin UI. Runs can be hidden (moved to a “Past Videos” section) without deletion.

Key concept: Each run is uniquely identified by its Unix timestamp (ts). All social posting state for a run is keyed by {article_id}_{run_ts} in the site_settings table.

2. Database Schema

Articles Table Columns

The articles table includes these narrated-video-specific columns:

ColumnTypeDescription

video_narrated_scriptTEXTJSON array of narration segments for the most recent run.

Hidden Runs

Hidden run timestamps are stored in site_settings under the key narrated_hidden_runs_{article_id} as a JSON array of integer timestamps.

Stale Status Cleanup

On application startup, any video_narrated_status values matching the pattern running% are reset to NULL. This prevents stale “in progress” states from persisting after a server crash or restart.

3. Video Generation Pipeline

The generation is triggered by a POST to the generate endpoint. The route validates prerequisites (OpenAI client configured, no generation already running, carousel images present), sets status to running:script, and spawns _narrated_video_worker(article_id, cfg) as a daemon thread. All exceptions within the worker are caught, logged, and written to video_narrated_status as error:<msg>.

Configuration Parameters (from POST body)

ParameterOptionsDefaultDetails
formatvertical, squareverticalVertical = 720×1280, Square = 720×720
voicealloy, echo, fable, onyx, nova, shimmeronyxOpenAI TTS voice selection
tts_modeltts-1, tts-1-hdtts-1Standard or high-definition TTS
script_lenshort, medium, longmediumShort = 20–40 words, Medium = 40–70 words, Long = 70–100 words per segment
kb_speedslow, medium, fastmediumKen Burns zoom factor per frame: Slow = 0.0010, Medium = 0.0015, Fast = 0.0025
crf18, 23, 2823H.264 CRF quality: 18 = high quality, 23 = balanced, 28 = smaller file
fps25, 3025Target output frames per second
speed0.5 – 3.01.35Playback speed factor applied after rendering. Clamped to 0.5–3.0 range

Step 1: GPT Narration Script

  • Model: gpt-4.1-mini
  • Response format: response_format: json_object
  • Sends slide descriptions (composed from carousel_prompts + carousel_punchlines) to GPT, requesting a JSON response: {"segments": [...]}
  • If GPT returns fewer segments than slides, missing segments are padded with the corresponding punchline text as fallback
  • Any segment containing fewer than 3 words is replaced with fallback text derived from the punchline

Step 2: TTS Audio Per Segment

  • API: OpenAI Audio Speech — _openai_client.audio.speech.create()
  • Model: configurable (default tts-1)
  • Voice: configurable (default onyx)
  • Response format: mp3
  • Output path: static/articles/{id}/video/audio/segment_{i}.mp3
  • Status updates to running:audio:{i}/{n} after each segment

Step 3: FFmpeg Binary Resolution

  1. First tries shutil.which("ffmpeg") to find a system-installed FFmpeg
  2. Falls back to imageio_ffmpeg.get_ffmpeg_exe() for the bundled binary

Step 4: Per-Clip Video Generation

Status updates to running:clip:{i}/{n}. Two rendering modes exist per slide:

Cinemagraph Mode

Used when a cinemagraph URL exists for the slide. The cinemagraph video is looped (stream_loop -1), scaled and cropped to the target dimensions, and muxed with the audio track. Uses libx264 ultrafast preset. Render FPS is fixed at 12.

Ken Burns Mode (Still Image)

Applies an animated zoom-and-pan effect to a still image. The zoom factor is calculated as min(1.0 + zoom_step * 200, 1.3), where zoom_step is derived from the kb_speed parameter. The image is scaled to zoom_factor times the target dimensions, then cropped with time-varying X/Y offsets creating a smooth pan from bottom-right to top-left. Uses libx264 ultrafast preset with -tune stillimage.

Audio Duration Probing

Audio duration is probed via ffprobe (with an ffmpeg -i fallback). The video clip duration is set to the audio duration plus 0.3 seconds of padding.

Step 5: Concatenation

Writes a concat.txt file listing all per-slide clips, then runs the FFmpeg concat demuxer:

ffmpeg -f concat -safe 0 -i concat.txt -c copy output.mp4

Status: running:final.

Step 5b: Speed Adjustment

If speed_factor != 1.0, the concatenated video is re-encoded with FFmpeg filters:

  • Video filter: setpts=PTS/{speed_factor}
  • Audio filter: atempo={speed_factor}

Step 6: Upload to GCS

  • GCS path: articles/{article_id}/video/narrated_{ts}.mp4
  • Falls back to a local static file URL if GCS is not configured
  • The generation log is also uploaded to GCS
  • A run history object is built (with ts, ts_label, url, params, segments, cinemagraph_slides) and prepended to the video_narrated_runs array
  • Legacy fields video_narrated_url and video_narrated_script are also updated
  • Status is set to NULL (idle) upon completion
  • Temporary files (audio segments, video clips, concat file) are deleted

5. Social Media Posting — Platform Details

Instagram (Reel)

Worker: _post_narrated_reel_worker

  1. Create Reel container via IG Graph API with media_type=REELS, passing video_url and caption
  2. Poll container status until FINISHED (5-minute deadline, 5–30 second polling intervals)
  3. Publish the container via the media_publish endpoint
  4. If publish returns an error, check recent posts to verify whether the Reel actually went through (fallback verification)
  5. Fetch permalink and store media_id in site_settings

Caption limit: 2,200 characters. Supports caption editing on live posts via the IG Graph API.

Facebook

Worker: _post_narrated_fb_worker

  • Posts via POST /{FB_PAGE_ID}/videos with the file_url parameter (Facebook fetches the video directly from GCS)
  • Single API call — no container/poll pattern needed
  • Supports delete via DELETE /{media_id}

YouTube Shorts

Worker: _upload_to_youtube_worker

Requires an OAuth refresh token stored in site_settings as youtube_refresh_token.

OAuth Flow

  1. GET /admin/youtube-connect → redirects to Google consent screen
  2. User authorises → callback to GET /admin/youtube-oauth-callback
  3. Refresh token stored in site_settings

Upload Steps

  1. Refresh the access token using the stored refresh token
  2. Download the video to a temporary file
  3. Perform a resumable upload to YouTube Data API v3
  4. Metadata: tags=["stamps", "philately", "shorts"], categoryId="26" (Howto & Style), privacyStatus="public", not made for kids

Result URL format: https://www.youtube.com/shorts/{video_id}

X (Twitter)

Worker: _post_narrated_x_worker

Auth: OAuth 1.0a via _x_auth()

  1. Download video to a temporary file
  2. INIT — Start chunked upload session
  3. APPEND — Upload chunks (5 MB each)
  4. FINALIZE — Complete the upload
  5. STATUS — Poll until "succeeded" (5-minute timeout)
  6. Create tweet with media_ids

Caption limit: 280 characters. Supports delete via DELETE /2/tweets/{tweet_id}.

X video duration limit: X imposes a 120-second video duration limit. The frontend checks duration and shows a warning instead of the post button if exceeded.

Threads

Worker: _post_narrated_threads_worker

  1. Create video container with media_type=VIDEO, passing video_url and text (truncated to 500 chars)
  2. Poll container status until FINISHED (5-minute deadline, 5-second intervals)
  3. Publish via threads_publish endpoint
  4. Fetch permalink

Caption limit: 500 characters.

Pinterest

Worker: _post_narrated_pinterest_worker

  1. Register media upload via POST /media with media_type=video
  2. Download video to a temporary file
  3. Upload to Pinterest’s S3 endpoint via multipart form (using upload_params from the registration response)
  4. Poll media status until "succeeded" (~7.5 minute timeout)
  5. Create video pin via POST /pins with board_id, title, description, media_source=video_id, and article link

Supports delete via DELETE /pins/{pin_id}.

TikTok

Worker: _post_narrated_tiktok_worker

  • Uses the Content Posting API with PULL_FROM_URL source
  • Initialises via POST /post/publish/video/init/ with post_info containing: title (150 char limit), privacy level PUBLIC_TO_EVERYONE, duet/comment/stitch enabled
  • Polls POST /post/publish/status/fetch/ until PUBLISH_COMPLETE (5-minute timeout)
TikTok caption limit: 150 characters. There is no API delete endpoint — deletion must be done directly on TikTok, then the post record can be archived locally.

LinkedIn

Worker: _post_narrated_linkedin_worker

Author: urn:li:organization:{LINKEDIN_ORG_ID}

  1. Download video to a temporary file
  2. Initialize upload via POST /videos?action=initializeUpload
  3. Upload binary via PUT request to the upload URL
  4. Poll video status until AVAILABLE (5-minute timeout)
  5. Create post via POST /posts with the video URN

Caption limit: 3,000 characters. Visibility: PUBLIC, distribution: MAIN_FEED. Supports delete.

6. Run Visibility Toggle

POST /admin/articles/<id>/toggle-narrated-run-visibility

Body: {"ts": <int>, "hidden": <bool>}

Adds or removes the given timestamp from the narrated_hidden_runs_{article_id} JSON array in site_settings. Hidden runs are moved from the main accordion to the “Past Videos” section in the frontend. The run data itself is not deleted.

Key JS Functions

buildRunItem(run, expandIt)

The core frontend function. NV runs are built dynamically via JS template literals. Builds a complete accordion item for a single run, including:

  • Header: Badges (format, voice, speed, cinemagraph count), timestamp, social media status icons (all 8 platforms), eye icon for hide/show toggle
  • Icons: IG, YT, FB, X use inline SVGs; Threads, Pinterest, TikTok, LinkedIn use Font Awesome brand icons
  • Body: Video Details collapse (params table), <video> element, Narration Script collapse (ordered list of segments), per-platform posting sections
  • Platform sections: Each wrapped in <div data-smc-net="..."> for network visibility toggling
  • Per-platform UI: Caption textarea + character counter + “Copy from Instagram” link + post button + status area + result area + Past Posts collapse
  • X section special: Duration check — if video exceeds 120 seconds, shows a warning instead of the post button

narratedGenerate()

Reads current settings from the modal, POSTs the config to the generate endpoint, and shows the running UI with progress indicators.

narratedStop()

POSTs to the stop endpoint, then calls narratedFinish() to reset the UI.

loadNarratedRuns(runs, hiddenTs)

Called on page load. Builds all accordion items, restores posting state for each platform on each run, fetches YouTube status, and applies SMC visibility preferences.

addNarratedRun(run)

Called after a successful generation. Collapses all existing accordion items, inserts the new run at the top (expanded), and applies SMC preferences. Caption auto-population occurs after run insertion.

hideNarratedRun(ts) / restoreNarratedRun(ts)

Moves the DOM accordion item between the main accordion and the Past Videos accordion. POSTs the visibility toggle to the server.

Polling & Status

_narratedApplyStatus(status, nSlides)

Maps the video_narrated_status string to a progress percentage:

Status PatternProgress
running:script5%
running:audio:N/M5% – 35% (proportional to N/M)
running:clip:N/M35% – 90% (proportional to N/M)
running:final90%

_narratedPoll()

Polls the /video-data endpoint every 20 seconds while generation is in progress. On completion (status is NULL and runs array has a new entry), calls addNarratedRun() to insert the new run.
