Narrated Video — Technical Documentation

Source: app.py, templates/article_edit.html Last updated: 2026-03-21

1. Overview

The narrated video system generates slideshow videos from article carousel images with AI-narrated audio. It uses GPT-4.1-mini for narration script generation, OpenAI TTS for audio synthesis, and FFmpeg for video rendering with Ken Burns effects or cinemagraph animation.

AttributeValue
Max slides10
Output formatsVertical (720×1280), Square (720×720)
Voicesalloy, echo, fable, onyx, nova, shimmer
TTS qualitytts-1 (standard), tts-1-hd (high definition)
Slide animationKen Burns (still images) or Cinemagraph (animated slides)
Social platformsInstagram Reels, YouTube Shorts, Facebook, X, Threads, Pinterest, TikTok, LinkedIn

Multiple generation runs are supported per article. Each run is stored as an object in a JSON array and rendered as an accordion item in the admin UI. Runs can be hidden (moved to a “Past Videos” section) without deletion.

Key concept: Each run is uniquely identified by its Unix timestamp (ts). All social posting state for a run is keyed by {article_id}_{run_ts} in the site_settings table.

Hidden Runs

Hidden run timestamps are stored in site_settings under the key narrated_hidden_runs_{article_id} as a JSON array of integer timestamps.

Stale Status Cleanup

On application startup, any video_narrated_status values that begin with running are reset to NULL. This prevents stale “in progress” states from persisting after a server restart.

3. Generation Pipeline

Generate Route

POST /admin/articles/<id>/generate-narrated-video

Accepts a JSON body with generation configuration parameters.

{
  "format":     "vertical",   // or "square"
  "voice":      "alloy",      // alloy|echo|fable|onyx|nova|shimmer
  "tts_model":  "tts-1",      // tts-1|tts-1-hd
  "script_len": "medium",     // short|medium|long
  "kb_speed":   "medium",     // slow|medium|fast
  "speed":      1.35,         // 0.5 - 3.0
  "crf":        23,           // 18|23|28
  "fps":        30            // 25|30
}

Validation:

  • OpenAI client must be configured (returns 500 if missing)
  • Article must not already have a generation in progress (returns 409)
  • Article must have carousel images (returns 400 if empty)

On success, sets status to running:script and spawns the _narrated_video_worker as a daemon thread.

Worker Thread

Function: _narrated_video_worker(article_id, cfg)

Runs as a daemon thread. All exceptions are caught, logged, and written to video_narrated_status as error:<msg>.

Configuration Parameters

ParameterOptionsDefaultDetails
formatvertical, squareverticalVertical = 720×1280, Square = 720×720
voicealloy, echo, fable, onyx, nova, shimmeralloyOpenAI TTS voice selection
tts_modeltts-1, tts-1-hdtts-1Standard or high-definition TTS
script_lenshort, medium, longmediumShort = 20–40 words, Medium = 40–70 words, Long = 70–100 words per segment
kb_speedslow, medium, fastmediumSlow = 0.0010, Medium = 0.0015, Fast = 0.0025 (Ken Burns pan/zoom factor per frame)

Pipeline Steps

Step 1: Fetch Article Data

Loads carousel_images, carousel_prompts, carousel_punchlines, title, and carousel_cinemagraphs from the database. Maximum 10 slides are used. Status remains running:script.

Step 2: GPT Narration Script

Calls GPT-4.1-mini with a system prompt that describes the slides (including prompts and punchlines). Expects a JSON response: {"segments": [...]}. Each segment is validated to contain at least 3 words. If GPT returns unusable data, the system falls back to using the carousel punchlines as narration text. Status: running:script.

Step 3: TTS Audio Generation

For each narration segment, calls _openai_client.audio.speech.create() with the configured model and voice. Audio files are saved as MP3 to static/articles/{id}/video/audio/segment_{i}.mp3. Status updates: running:audio:{i}/{n}.

Step 4: Per-Clip Video Rendering

Status updates: running:clip:{i}/{n}. Two rendering paths exist:

Cinemagraph path: If a cinemagraph video exists for the slide, loops it with stream_loop -1, scales and crops to the target dimensions, and overlays the audio track. Uses libx264 ultrafast preset.

Ken Burns path: For still images, applies an animated zoom-and-pan effect. The image is scaled to zoom_factor times the target dimensions, then cropped with time-varying X/Y offsets to create smooth pan motion. Uses libx264 ultrafast preset with tune=stillimage.

The render_fps is fixed at 12 internally regardless of the output fps setting. The output FPS is achieved during the final encoding stage.

Step 5: Concatenation

Writes a concat.txt file listing all per-slide clips, then runs:

ffmpeg -f concat -safe 0 -i concat.txt -c copy output.mp4

Status: running:final.

Step 6: Speed Adjustment

If speed_factor != 1.0, applies FFmpeg filters:

  • Video: setpts=PTS/{factor}
  • Audio: atempo={factor}

Step 7: GCS Upload

Uploads the final video to GCS at path articles/{id}/video/narrated_{ts}.mp4. Falls back to a local static file path if GCS is not configured.

Step 8: Save Run History

Builds a run object with ts, ts_label, url, params, segments, and cinemagraph_slides. Prepends it to the runs array. Also updates the legacy fields video_narrated_url and video_narrated_script. Sets status to NULL.

Step 9: Cleanup

Persists the generation log to the database (capped at 65 KB) and uploads it to GCS. Writes an activity log entry. Deletes all temporary files (audio segments, video clips, concat file).

Stop Route

POST /admin/articles/<id>/stop-narrated-video

Sets video_narrated_status to NULL. The worker thread continues running in the background but the frontend stops polling and ignores any further updates.

4. Run Visibility

POST /admin/articles/<id>/toggle-narrated-run-visibility

Body: {"ts": <int>, "hidden": <bool>}

Adds or removes the given timestamp from the narrated_hidden_runs_{article_id} JSON array in site_settings. Hidden runs are moved from the main accordion to the “Past Videos” section in the frontend.

5. Social: Instagram Reels

Worker: _post_narrated_reel_worker

  1. Create Reel container via IG Graph API with media_type=REELS
  2. Poll container status until FINISHED (5-minute deadline, 5–30 second intervals)
  3. Publish via media_publish endpoint
  4. If an error occurs during publish, verify by checking recent posts
  5. Fetch permalink and store media_id

8. Social: X (Twitter)

Worker: _post_narrated_x_worker

  1. Download video to a temporary file
  2. INIT — Start chunked upload session
  3. APPEND — Upload chunks (5 MB each)
  4. FINALIZE — Complete the upload
  5. STATUS — Poll until "succeeded" (5-minute timeout)
  6. Create tweet with media_ids

Uses OAuth 1.0a authentication via _x_auth().

X video duration limit: X imposes a 120-second video duration limit. The frontend checks duration and shows a warning instead of the post button if exceeded.

Key JS Functions

buildRunItem(run, expandIt)

The largest frontend function. Builds a complete accordion item for a single run, including:

  • Header: Badges (format, voice, speed, cinemagraph count), timestamp, social media status icons (8 platforms), eye icon for hide/show toggle
  • Icons: IG, YT, FB, X use inline SVGs; Threads, Pinterest, TikTok, LinkedIn use Font Awesome brand icons
  • Eye icon: Uses an eye-off SVG, triggers hide/show toggle
  • Body: Video Details collapse (params table), <video> element, Narration Script collapse (ordered list of segments), per-platform posting sections
  • Platform sections: Each wrapped in <div data-smc-net="..."> for network visibility toggling
  • Per-platform UI: Caption textarea + character counter + “Copy from Instagram” link + post button + status area + result area + Past Posts collapse
  • X section special: Duration check; if video exceeds 120 seconds, shows a warning instead of the post button

narratedGenerate()

Reads current settings from the modal, POSTs the config to the generate endpoint, and shows the running UI with progress indicators.

narratedStop()

POSTs to the stop endpoint, then calls narratedFinish() to reset the UI.

loadNarratedRuns(runs, hiddenTs)

Called on page load. Builds all accordion items, restores posting state for each platform on each run, fetches YouTube status, and applies SMC (Social Media Content) visibility preferences.

addNarratedRun(run)

Called after a successful generation. Collapses all existing accordion items, inserts the new run at the top (expanded), and applies SMC preferences.

hideNarratedRun(ts) / restoreNarratedRun(ts)

Moves the DOM accordion item between the main accordion and the Past Videos accordion. POSTs the visibility toggle to the server.

Polling & Status Mapping

_narratedApplyStatus(status, nSlides)

Maps the video_narrated_status string to a progress percentage:

Status PatternProgress

_narratedPoll()

Polls the /video-data endpoint every 20 seconds while generation is in progress. On completion, calls addNarratedRun() to insert the new run.

14. GCS Storage Structure

articles/{article_id}/video/
  narrated_{timestamp}.mp4            — Final output video
  audio/segment_{N}.mp3               — TTS audio per slide (cleaned up after generation)
  clips/clip_{N}.mp4                  — Per-slide video clips (cleaned up after generation)
  concat.txt                          — FFmpeg concat list (cleaned up after generation)
  narrated_run_{timestamp}.log        — Generation log file
Temporary files (audio segments, video clips, concat file) are deleted after successful generation. Only the final .mp4 and .log files persist in GCS.

17. Network Visibility Toggle

All platform posting sections in both carousel/cinemagraph and narrated video accordions are wrapped in <div data-smc-net="PLATFORM_KEY"> elements.
