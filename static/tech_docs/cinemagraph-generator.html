Cinemagraph Generator — Technical Documentation

Source: app.py, templates/article_edit.html Last updated: 2026-03-22

1. Overview

The cinemagraph generator creates animated 5-second looping video clips from static carousel images using Luma AI (ray-2 model). Each carousel slide can have a corresponding cinemagraph clip with subtle, atmospheric motion — producing a cinematic still-life effect suitable for social media.

AttributeValue
Max slides10 (one cinemagraph per carousel slot)
Video duration5 seconds, looping
Aspect ratio1:1 (square)
AI modelLuma AI ray-2
Output formatMP4
Generation time~30–90 seconds per slide (varies with Luma queue)

2. Database Schema

Articles Table

All cinemagraph data is stored on the articles table alongside the parent article.

ColumnTypeDescription
carousel_cinemagraphs JSON Array of 10 slots. Each slot is either an MP4 URL string (GCS or local /static/... path) or null if no cinemagraph has been generated for that slide.
carousel_cinemagraph_prompts JSON Array of 10 per-slide prompt strings. Each slot can be null (falls back to article-level or global prompt) or a custom prompt for that specific slide.
carousel_cinemagraph_created_at JSON Array of 10 ISO 8601 timestamp strings. Records when each cinemagraph clip was generated. null for empty slots.
carousel_cinemagraph_log TEXT Full-text generation log for the most recent batch run. Truncated to 65 KB maximum to stay within MySQL TEXT column limits.
carousel_cinemagraph_archived JSON Array of archived clip objects. Each object contains: url (MP4 URL), prompt (prompt used), created_at (original creation timestamp), archived_at (when archived), slot_source (original slot index 0–9).

Archived Clip Object Shape

{
  "url": "https://storage.googleapis.com/.../slide_3_1711234567.mp4",
  "prompt": "Subtle gentle atmospheric motion...",
  "created_at": "2026-03-20T14:30:00Z",
  "archived_at": "2026-03-21T09:15:00Z",
  "slot_source": 3
}

Site Settings Keys

The site_settings table stores runtime state and configuration using key-value patterns.

Key PatternExample ValueDescription
cinemagraph_prompt (prompt text) Global default prompt used for all cinemagraph generation when no per-article or per-slide prompt is set.
cinemagraph_prompt_{article_id} (prompt text) Per-article prompt override. Takes precedence over the global prompt but is overridden by per-slide prompts.
cinemagraph_status_{article_id} running:2/5 Worker progress indicator. Format: running:{completed}/{total}. Polled by the frontend to show progress.
cinemagraph_result_{article_id} done:3 or error:Billing limit Final outcome of the last generation run. done:{count} for success, error:{message} for failure.
cinemagraph_cancel_{article_id} 1 Cancel signal. Set to "1" to request cancellation of an in-progress generation. Checked by the worker before each slide and during polling.
ig_post_* (various) Instagram post state keys (shared pattern with carousel posting).
linkedin_* (various) LinkedIn post state keys (shared pattern with carousel posting).

3. Luma AI Integration

Cinemagraph generation uses the Luma AI Python SDK (lumaai package) to create short video clips from static images.

from lumaai import LumaAI
client = LumaAI(auth_token=os.environ["LUMAAI_API_KEY"])

_luma_create_task(image_url, prompt_text)

Creates a new Luma generation task from a source image and motion prompt.

ParameterValue
model"ray-2"
keyframes{"frame0": {"type": "image", "url": image_url}}
aspect_ratio"1:1"
duration"5s"
gen = client.generations.create(
    model="ray-2",
    prompt=prompt_text,
    keyframes={
        "frame0": {"type": "image", "url": image_url}
    },
    aspect_ratio="1:1",
    duration="5s"
)
return gen.id
The frame0 keyframe anchors the video to the source carousel image, ensuring the cinemagraph closely resembles the original still.

_luma_poll_task(gen_id)

Polls a Luma generation task for completion. Returns a tuple of (state, mp4_url, failure_reason).

StateMeaning
queuedTask is waiting in Luma's queue
dreamingVideo is being generated
completedVideo is ready; mp4_url is populated
failedGeneration failed; failure_reason is populated
gen = client.generations.get(gen_id)
state = gen.state
mp4_url = gen.assets.video if state == "completed" else None
failure_reason = gen.failure_reason if state == "failed" else None
return (state, mp4_url, failure_reason)
Poll interval is 6 seconds between checks in the worker loop.

Billing Error Detection

_is_luma_billing_error(exc) inspects exceptions to detect non-transient billing/credit errors. When detected, the worker aborts immediately rather than retrying, since billing errors will not resolve on their own.

Detected patterns include:

  • Insufficient credits
  • Billing-related error messages
  • HTTP 403 Forbidden responses

4. Prompt Resolution

When generating a cinemagraph for slide i, the prompt is resolved in priority order:

  1. Per-slide prompt — carousel_cinemagraph_prompts[i] (from the article’s JSON array)
  2. Per-article prompt — site_settings key cinemagraph_prompt_{article_id}
  3. Global prompt — site_settings key cinemagraph_prompt
  4. Hardcoded default — _CINE_DEFAULT_PROMPT

Default Prompt

_CINE_DEFAULT_PROMPT = "Subtle gentle atmospheric motion, cinematic still life, minimal movement, soft breathing effect, loop-friendly"
The resolution chain short-circuits at the first non-empty value. A per-slide prompt of null or empty string falls through to the next level.

5. Video Storage

File Paths

Generated MP4 files follow a consistent naming convention using the slide index and a Unix timestamp.

ContextPath Pattern
Local file static/articles/{article_id}/cinemagraph/slide_{N}_{timestamp}.mp4
GCS object articles/{article_id}/cinemagraph/slide_{N}_{timestamp}.mp4
Run log (GCS) articles/{article_id}/cinemagraph/run_{timestamp}.log

GCS Fallback Logic

The storage flow for each generated clip:

  1. Download MP4 from Luma’s temporary URL (timeout: 120 seconds)
  2. Save to local filesystem at static/articles/{article_id}/cinemagraph/
  3. Attempt upload to GCS via upload_bytes_to_gcs()
  4. If GCS succeeds: delete the local file, store the GCS URL in the database
  5. If GCS fails: keep the local file, store the /static/... path in the database
The database is updated after each individual slide, not batched at the end of the run. This ensures partial progress is preserved if the worker is interrupted.

Run logs are also uploaded to GCS after the batch completes, stored at the run_{timestamp}.log path.

6. API Routes

Generation Routes

MethodRouteDescription
POST /admin/articles/<id>/generate-cinemagraphs Starts batch generation for all empty (null) slots. Launches _cinemagraph_worker in a background thread. Returns immediately with status.
POST /admin/articles/<id>/cancel-cinemagraphs Sets the cancel signal (cinemagraph_cancel_{id} = "1") to request graceful cancellation of an in-progress generation.
POST /admin/articles/<id>/regenerate-cinemagraph-slide Body: {"slide_idx": N}. Archives the existing clip in the target slot (if any), then regenerates a new cinemagraph for that single slide. Launches _cinemagraph_slide_worker.

Archive & Restore Routes

MethodRouteDescription
POST /admin/articles/<id>/archive-cinemagraph-slide Body: {"slide_idx": N}. Moves the clip at the given slot to the archived array with metadata, then nulls the slot.
POST /admin/articles/<id>/restore-cinemagraph-clip Body: {"archived_idx": N, "target_slot": M}. Restores an archived clip to the target slot. If the target slot already has a clip, that clip is auto-archived first.
POST /admin/articles/<id>/archive-all-cinemagraphs Archives all non-null slots (iterates all 10 slots). Useful for clearing the board before a fresh generation run.

7. Background Workers

Batch Worker: _cinemagraph_worker()

Signature: _cinemagraph_worker(article_id, images, prompts, slot_indices, existing_urls)

Generates cinemagraphs for all empty slots in a single batch run. Launched in a background thread by the generate route.

Batch Worker Flow

  1. Set status to running:0/{total}
  2. For each empty slot index in slot_indices:
    1. Check cancel signal — abort if _is_cancelled() returns true
    2. Resolve prompt for this slide (see Prompt Resolution)
    3. Call _luma_create_task(image_url, prompt) to start generation
    4. Poll with _luma_poll_task(gen_id) every 6 seconds until completed or failed
    5. On completion: download MP4, upload to GCS, update DB
    6. Update status to running:{completed}/{total}
  3. After all slides: set result to done:{count}
  4. Upload run log to GCS
The worker aborts after 3 consecutive failures. This prevents runaway costs if Luma is experiencing persistent issues. The result is set to an error message indicating the abort reason.

Single Slide Worker: _cinemagraph_slide_worker()

Signature: _cinemagraph_slide_worker(article_id, slide_idx, img_url, prompt)

Regenerates a single slide. Follows the same create → poll → download → upload → DB update flow as the batch worker, but only for one slot. Used by the regenerate-slide route.

Cancel Support

Cancellation is cooperative, not preemptive. The cancel signal is stored in site_settings as cinemagraph_cancel_{article_id} = "1".

The worker checks _is_cancelled() at two points:

  • Before each slide — skips remaining slides if cancelled
  • During polling — abandons the current Luma task if cancelled mid-generation
Cancellation does not revoke in-progress Luma tasks on their servers. It only stops the local worker from polling and processing further. Already-completed slides are preserved.

8. Social Media Posting

Instagram (Animated Carousel)

Cinemagraph clips are posted to Instagram as an animated carousel where each slide is a video container.

Worker

_post_to_instagram_worker(article_id, caption, post_type="cine")

Instagram Posting Flow

  1. For each slide that has a cinemagraph clip:
    1. Generate overlay PNG via _make_cinemagraph_overlay_png()
    2. Composite overlay onto video via FFmpeg: _apply_overlay_to_video()
    3. Upload composed video to GCS
    4. Create IG child container with media_type=VIDEO, is_carousel_item=true
  2. For slides without a cinemagraph → fall back to static image container
  3. Poll all child containers until ready
  4. Create carousel container referencing all children
  5. Publish the carousel
  6. Fetch and store the permalink

Overlay Generation

_make_cinemagraph_overlay_png(
    punchline,          # text to display on the overlay
    slide_index,        # 0-based slide index
    total_slides,       # total number of slides
    width=960,          # output width in pixels
    height=960,         # output height in pixels
    band_top=None       # optional top position for text band
)

Generates a transparent PNG with the punchline text and a swipe-hint indicator. This PNG is then composited onto the video.

FFmpeg Compositing

_apply_overlay_to_video(video_url, overlay_png, tmp_mp4)

Overlays the generated PNG onto the MP4 video using FFmpeg’s overlay filter. The output is written to tmp_mp4 as a new MP4 file ready for Instagram upload.

LinkedIn (Static Image Carousel)

Worker

_post_carousel_linkedin_worker(article_id, caption, component="cine")

LinkedIn’s carousel API does not support video. Cinemagraph slides are posted as static composed images using compose_carousel_slide(), the same compositing function used for regular carousel posting.

LinkedIn Posting Flow

  1. For each slide: compose a static image via compose_carousel_slide()
  2. Initialize multipart upload with LinkedIn API
  3. PUT image bytes for each slide
  4. Create post with multiImage.images payload

9. Archive & Delete Flow

Archive Single Slide

  1. Read the clip URL, prompt, and created_at from the current slot
  2. Append an archived clip object (with url, prompt, created_at, archived_at, slot_source) to carousel_cinemagraph_archived
  3. Set the slot to null in carousel_cinemagraphs
  4. Update the database

Archive All

Iterates all 10 slots and archives every non-null clip using the same single-slide archive logic.

Restore Archived Clip

  1. Read the archived clip at archived_idx
  2. If target_slot already has a clip, auto-archive it first (preventing data loss)
  3. Place the restored clip’s URL into target_slot
  4. Remove the clip from the archived array
  5. Update the database

Social Post Archive & Delete

  • Instagram archive: Same pattern as carousel — moves post record to a history array in site_settings
  • LinkedIn archive: Moves the post record to a history array
  • LinkedIn delete: Calls the LinkedIn DELETE API to remove the post from the platform, then archives the record locally

10. Frontend Architecture

Template Structure

All cinemagraph UI lives in templates/article_edit.html, within the cinemagraph tab/panel section.

Social Media Network Wrappers

Each social platform section is wrapped in a data-smc-net attribute for JavaScript targeting:

data-smc-net valueContent
igFull posting UI: caption textarea, post button, status area, post history
linkedinFull posting UI: caption textarea, post button, status area, post history
fbUnsupported notice (alert-info)

threadsUnsupported notice (alert-info)
pinterestUnsupported notice (alert-info)

Status Polling

The frontend uses setInterval to poll generation and posting status:

  • Generation status: polls /admin/articles/<id>/video-data to check cinemagraph_status_{id} and cinemagraph_result_{id}
  • Instagram posting status: polls /admin/articles/<id>/ig-post-status
  • LinkedIn posting status: polls /admin/articles/<id>/carousel-linkedin-status

Post-Success Behavior

_makeViewPostBtn() transforms the post button into a “View Post” link after a successful social media post, linking to the published content on the platform.

Unsupported Platform Notices

Unsupported platforms (Facebook, X, Threads, Pinterest) render a Bootstrap alert-info div explaining that the platform does not support cinemagraph-style looping video carousels via its API. No posting controls are displayed for these platforms.
