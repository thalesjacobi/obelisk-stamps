Cinemagraph Generator

File: app.py Template: article_detail.html Last updated: 2026-03-21

1. Overview

The Cinemagraph Generator converts static carousel images into 5-second looping video clips using the Luma AI Ray-2 model. Each clip adds subtle atmospheric motion to a still image, producing a cinemagraph effect suitable for social media posts.

Key characteristics:
  • Requires at least 1 existing carousel image to unlock
  • Produces up to 10 clips (one per carousel slide)
  • Output: MP4, 1:1 aspect ratio, 5 seconds, autoplay / loop / muted
  • Cost: approximately $0.25 per clip (Luma AI pricing)
  • Supports per-slide custom prompts for fine-grained control

4. Video Overlay Functions

Overlay PNG Creation

_make_cinemagraph_overlay_png

_make_cinemagraph_overlay_png(
    punchline,
    slide_index,
    total_slides,
    width=960,
    height=960,
    band_top=None
)

Creates a transparent PNG overlay with a dark text band and punchline text, using the same visual style as carousel image compositing. The band is positioned consistently across all slides via band_top.

_compute_max_overlay_band_top

_compute_max_overlay_band_top(punchlines, ...)

Calculates a consistent band position across all slides so that the overlay band appears at the same vertical position in every clip of the carousel.

FFmpeg Compositing

_apply_overlay_to_video

_apply_overlay_to_video(video_url_or_path, overlay_png_bytes, output_path)

FFmpeg pipeline:

  1. Scale video to 960x960
  2. Overlay the transparent PNG
  3. Encode with libx264, CRF 20
  4. No audio stream
  5. Pixel format: yuv420p
  6. Timeout: 120 seconds

5. Generation Pipeline

Route

POST /admin/articles/<id>/generate-cinemagraphs

Validation checks:

  1. _luma_enabled must be true (API key configured)
  2. No generation already running for this article
  3. Article must have at least one carousel image

Startup sequence:

  1. Loads carousel images (up to 10), existing prompts, existing clips
  2. Identifies empty slots (have source image but no clip); skips occupied slots
  3. Saves prompts to DB, clears stale result key
  4. Sets status to running:0/{n_empty}
  5. Spawns _cinemagraph_worker as a daemon thread
  6. Returns {"started": true, "n_slides": n_empty}

Worker: _cinemagraph_worker

_cinemagraph_worker(article_id, images, prompts, slot_indices, existing_urls)

For each empty slot the worker performs:

  1. Check cancel flag in site_settings
  2. Set status to running:{i}/{n}
  3. Resolve image URL — GCS URL used directly; local images are temp-uploaded to GCS
  4. Resolve prompt via the prompt resolution order
  5. Call _luma_create_task() to get a generation ID
  6. Poll _luma_poll_task() every 6 seconds, updating status sub-state (queued / dreaming)
  7. On completion: download MP4, save locally, upload to GCS, clean local copy if GCS succeeds
  8. Update carousel_cinemagraphs and carousel_cinemagraph_created_at in DB immediately

Abort conditions

  • Cancel flag detected (cinemagraph_cancel_{article_id} = "1")
  • Billing error detected via _is_luma_billing_error()
  • 3 consecutive failures

Cleanup

  • Sets result: done:{count} or error:...
  • Clears status and cancel keys from site_settings
  • Persists run log to DB (truncated to 64 KB max) and to GCS

6. Single Slide Operations

Regenerate Slide

POST /admin/articles/<id>/regenerate-cinemagraph-slide

Body fieldTypeDescription
slide_idxintZero-based slide index to regenerate
promptstringLuma prompt for this slide

Auto-archives the existing clip in that slot before starting. Sets status to slide_running:{slide_idx} and spawns _cinemagraph_slide_worker as a daemon thread.

Archive Slide

POST /admin/articles/<id>/archive-cinemagraph-slide

Body fieldTypeDescription
slide_idxintZero-based slide index to archive

Appends clip metadata to the archived array and nullifies the slot.

Restore Archived Clip

POST /admin/articles/<id>/restore-cinemagraph-clip

Body fieldTypeDescription
archived_idxintIndex within the archived array
target_slotintDestination slot in the active array

Supports both legacy string entries and new dict entries in the archive. If the target slot is occupied, the displaced clip is auto-archived before restoration.

Archive All

POST /admin/articles/<id>/archive-all-cinemagraphs

Moves all non-null clips to the archived array and clears all active slots.

Cancel Generation

POST /admin/articles/<id>/cancel-cinemagraphs

Sets the cancel flag (cinemagraph_cancel_{article_id} = "1").

Clear Result

DELETE /admin/articles/<id>/cinemagraph-result

Deletes the result key to prevent an error message from reappearing on page reload.

7. Social Media Posting — Instagram

POST /admin/articles/<id>/post-to-instagram

Body fieldTypeDescription
captionstringPost caption text
typestringMust be "cine"

Worker flow

  1. Creates a mixed carousel: cinemagraph clips as VIDEO containers, static images as IMAGE fallback for slots without clips
  2. Each cinemagraph gets a punchline overlay via _apply_overlay_to_video()
  3. Uploads composited MP4s to GCS (as ig_composed_{N}_{timestamp}.mp4)
  4. Creates IG child containers for each media item
  5. Polls each container until FINISHED
  6. Creates the carousel container and publishes
Status check, edit, and archive operations follow the same pattern as carousel posting but use the type=cine parameter to distinguish cinemagraph posts from static image posts.

8. Social Media Posting — LinkedIn

POST /admin/articles/<id>/post-carousel-to-linkedin

Body fieldTypeDescription
captionstringPost caption text
typestringMust be "cine"
Important: LinkedIn receives static images with punchline overlays even when the type is "cine". Cinemagraph video clips are not used for LinkedIn — the worker composes static carousel images with punchline overlays instead.

Upload flow follows the standard LinkedIn carousel pipeline:

  1. initializeUpload — request upload URL from LinkedIn API
  2. PUT image bytes to the upload URL
  3. Create the post with the uploaded asset references

9. Unsupported Platforms

The following platforms display informational notices in the admin UI explaining that cinemagraph-style looping videos are not supported via their APIs:

  • Facebook
  • X (Twitter)
  • Threads
  • Pinterest

Users are directed to use Carousel Images or Narrated Videos instead for these platforms.

12. Prompt Resolution Order

When resolving the Luma prompt for a given slide, the system checks the following sources in order, using the first non-empty value:

  1. Per-slide prompt — carousel_cinemagraph_prompts[i] from the articles table
  2. Per-article override — cinemagraph_prompt_{article_id} in site_settings
  3. Global setting — cinemagraph_prompt in site_settings
  4. Hardcoded default — _CINE_DEFAULT_PROMPT
The hardcoded default is: "Subtle gentle atmospheric motion, cinematic still life, minimal movement, soft breathing effect, loop-friendly"

13. Integration with Narrated Video

When the narrated video worker builds per-slide segments, it checks whether cinemagraphs[i] exists for the current slide. If a cinemagraph MP4 is available, it is used as the video base instead of the static Ken Burns animation.

The cinemagraph clip is integrated as follows:

  • Looped with stream_loop -1 to ensure sufficient duration
  • Trimmed to match the audio segment duration for that slide
