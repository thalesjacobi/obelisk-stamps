Carousel Generator — Technical Documentation

Source: app.py, templates/article_edit.html Last updated: 2026-03-21

1. Overview

The carousel generator creates up to 10 square images via DALL-E 3 from article content. Each image can have a punchline (text overlay) and a per-slide DALL-E prompt. Generated images are then available for posting to Instagram, Facebook, X, Threads, Pinterest, and LinkedIn.

AttributeValue
Max slides10
Generation time~60–90 seconds
Cost per run~$0.40
DALL-E native size1024×1024
Composed output size960×960 (with punchline text overlays)
Images are generated at 1024×1024 by DALL-E 3 and then composed to 960×960 with punchline text overlays before posting to social platforms.

3. Image Compositing

Two key functions in app.py handle compositing punchline text overlays onto generated images.

compose_carousel_slide()

compose_carousel_slide(img_bytes, punchline, slide_index, total_slides, width=960, height=960, band_top=None)
  • Takes raw image bytes and composites a punchline text overlay
  • Dark band at the bottom (~92% opacity) with gradient fade above
  • White uppercase text, word-wrapped, left-aligned
  • Yellow "Slide right for more >>>" hint on all slides except the last
  • Uses Pillow for rendering
  • Font: _FONT_PATH (embedded font file)
  • Font sizes: 4.8% of width (main punchline text), 2.1% of width (slide hint)
  • Returns composed JPEG bytes
The band_top parameter controls the Y position where the dark overlay band begins. When None, it is computed per-slide. Pass a pre-computed value for consistent band placement across all slides.

_compute_max_overlay_band_top()

_compute_max_overlay_band_top(punchlines, width=960, height=960)
  • Pre-computes a consistent Y position for the text band across all slides
  • Iterates all punchlines, calculates the required band height for each
  • Returns the minimum band_top value (i.e., the tallest band needed)
  • Ensures all slides have identical band placement regardless of text length

4. Generation Pipeline

Generate Route

POST /admin/articles/<id>/carousel/generate

  1. Validates: OpenAI client is initialized, generation is not already running, article exists
  2. Reads article title, body content, existing prompts and punchlines from DB
  3. Sets status to running:0/{n} in site_settings
  4. Spawns _carousel_worker as a daemon thread

Worker: _carousel_worker(article_id, ...)

For each slide (up to 10):

  1. Checks cancel flag (carousel_cancel_{article_id})
  2. Generates DALL-E prompt via GPT-4.1-mini if no prompt provided for this slide
  3. Calls DALL-E 3 API: model="dall-e-3", size="1024x1024", quality="standard", n=1
  4. Downloads generated image from the returned URL
  5. Saves locally to static/articles/{id}/carousel/slide_{i}_{ts}.jpg
  6. Uploads to GCS: articles/{id}/carousel/slide_{i}_{ts}.jpg
  7. Updates DB immediately (appends to carousel_images JSON array)
  8. Updates status: running:{i}/{n}

On Completion

  • Sets result to done:{count}, clears status
  • Writes activity log entry

On Cancel

  • Sets result to cancelled, clears status
Generation costs ~$0.04 per image ($0.40 for a full 10-slide carousel). The worker runs in a daemon thread and updates the DB after each individual slide completes, so partial results are preserved if the process is interrupted.

5. Image Management Routes

Rerun Single Slide

POST /admin/articles/<id>/carousel/rerun

  • Body: {slide_idx, prompt}
  • Archives the existing image for that slot
  • Generates a new image using the provided prompt
  • Runs as a single-slide background worker

Archive One Slide

POST /admin/articles/<id>/carousel/archive-slide

  • Body: {slide_idx}
  • Moves image to carousel_archived array with metadata: url, prompt, punchline, archived_at, slot_source

Restore Archived Image

POST /admin/articles/<id>/carousel/restore

  • Body: {archived_idx, target_slot}
  • Restores from carousel_archived array to a specific slot (0–9)
  • Auto-archives the displaced image if the target slot is already occupied

Archive All

POST /admin/articles/<id>/carousel/archive-all

  • Moves all non-null images to the archived array
  • Clears all image slots

Cancel Generation

POST /admin/articles/<id>/carousel/cancel

  • Sets cancel flag in site_settings
  • Worker checks the flag before generating each slide

Clear Result Notification

DELETE /admin/articles/<id>/carousel/result

  • Deletes the carousel_result_{id} key from site_settings

6. Caption Generation

POST /admin/articles/<id>/generate-ig-caption

  • Body: {component: "carousel"|"cinemagraph"|"narrated"}
  • Uses GPT-4.1-mini to generate an Instagram caption from article content
  • System prompt instructs: engaging caption, relevant hashtags, no URLs
  • Returns: {caption: "..."}
  • Cost: ~$0.01 per generation

7. Social Media Posting — Instagram

Post Route

POST /admin/articles/<id>/post-to-instagram

  • Body: {caption, type: "car"|"cine"}
  • For type="car": requires at least 2 carousel images
  • Spawns _post_to_instagram_worker

Worker Flow (type="car")

  1. Loads carousel images and punchlines from DB
  2. Computes consistent band_top for overlays via _compute_max_overlay_band_top()
  3. For each slide: composes overlay via compose_carousel_slide(), uploads composed JPEG to GCS as ig_composed_{N}_{ts}.jpg
  4. Creates IG child containers (IMAGE type) via Graph API
  5. Polls each child container until FINISHED (5-minute timeout)
  6. Creates carousel container with all children
  7. Polls carousel container until FINISHED
  8. Publishes via POST /{ig_user_id}/media_publish
  9. Fetches permalink
  10. Stores media_id, permalink, caption snapshot in DB

Supporting Routes

/admin/articles/<id>/check-instagram-post?type=carGETCheck if post is still live (queries GET /{media_id}?fields=id,permalink)
/admin/articles/<id>/edit-instagram-captionPOSTEdit caption (body: {caption, type}, calls POST /{media_id})
/admin/articles/<id>/archive-instagram-postPOSTMove to history array, clear active keys

9. Social Media Posting — X (Twitter)

Post Route

POST /admin/articles/<id>/post-carousel-to-x

  • Body: {caption}
  • Spawns _post_carousel_x_worker

Worker Flow

  1. Composes each slide with overlay (max 4 images for X)
  2. For each image: INIT chunked upload, APPEND chunks (5 MB), FINALIZE, poll STATUS
  3. Creates tweet with media_ids array via X API v2
  4. Stores tweet_id
X limits media attachments to 4 images per tweet. Only the first 4 carousel slides are posted.

11. Social Media Posting — Pinterest

Post Route

POST /admin/articles/<id>/post-carousel-to-pinterest

  • Body: {caption}
  • Spawns _post_carousel_pinterest_worker

Worker Flow

  1. Composes the first slide as the pin image
  2. Uploads image bytes to Pinterest via POST /pins
  3. Uses caption as pin title (first 100 chars) and description (first 800 chars)
  4. Links to article URL if available
  5. Stores pin_id
Pinterest posts only the first slide as a single pin, not a multi-image carousel. Title is truncated to 100 characters, description to 800 characters.

13. Frontend Architecture

HTML Structure (article_edit.html)

  • Card: Social Media Content panel
  • Tabs: Images Grid (count badge) | Archived (count badge) | Preview
  • Image grid: flexbox layout, 5 columns (20% width), responsive (3 columns at 768px)
  • Each image has: click to open edit modal, "i" info button
  • Per-platform sections wrapped in <div data-smc-net="PLATFORM"> for visibility toggle

Network Visibility Toggle

  • Horizontal kebab menu (⋯) in card header with checkboxes per network
  • Preferences stored in localStorage under key smcNetworkPrefs
  • Defaults: all enabled except TikTok (disabled by default)

Polling Pattern (shared across all platforms)

  1. POST to platform route — returns immediately
  2. Frontend polls the status endpoint every 5 seconds
  3. Status transitions: null → running:... → done:permalink or error:message
  4. On done: — transforms post button to "Already posted, click to view" with platform icon
  5. On error: — shows dismissible Bootstrap alert

Copy from Instagram & Caption Auto-Population

Copy from Instagram

  • Each non-Instagram platform has a "Copy from Instagram" link
  • Copies the IG caption textarea value to the target platform's textarea
  • Truncates to platform-specific character limit

Caption Auto-Population

  • On page load (DOMContentLoaded), fills empty caption fields for Threads, Pinterest, LinkedIn, and X with defaultCaption (article title + URL)
  • Instagram caption is NOT auto-filled (has a separate "Generate caption" link)

14. GCS Storage Structure

articles/{article_id}/carousel/
  slide_{N}_{timestamp}.jpg              — raw DALL-E output
  ig_composed_{N}_{timestamp}.jpg        — with punchline overlay for Instagram
  fb_composed_{N}_{timestamp}.jpg        — with punchline overlay for Facebook
  x_composed_{N}_{timestamp}.jpg         — with punchline overlay for X
  threads_composed_{N}_{timestamp}.jpg   — with punchline overlay for Threads
  pinterest_composed_{N}_{timestamp}.jpg — with punchline overlay for Pinterest
  linkedin_composed_{N}_{timestamp}.jpg  — with punchline overlay for LinkedIn

Local fallback: static/articles/{article_id}/carousel/slide_{N}_{ts}.jpg

Each platform gets its own composed image file because overlay composition happens at posting time. The raw DALL-E output is stored separately and reused across platforms.

15. Settings

  • Carousel Image Style: a global DALL-E style prompt stored in site_settings key carousel_image_style
  • Configurable at /admin/settings under "Carousel Image Style"
  • Appended to every DALL-E prompt when generating carousel images
  • Changes take effect on the next carousel generation

17. Error Handling

  • All posting workers catch exceptions and set error:message in site_settings
  • Frontend displays errors as dismissible Bootstrap alerts
  • Rate limit handling: Instagram worker checks recent posts to confirm publication even if the API returns an error
  • Stale status keys are purged on server startup

18. Icon System

  • Section headers: use inline SVGs (Instagram camera, Facebook F, X logo) or Font Awesome brand icons (Threads, Pinterest, LinkedIn)
  • Post buttons inside body containers: use FA brand icons: <i class="fa-brands fa-{platform}">
  • "Already posted" buttons: use the _platformIconSvg map (FA brand icons) + _extLinkSvg (inline SVG external link icon)
CSS fix: font-family: "Font Awesome 6 Brands" !important override on .fa-brands elements inside body containers. This is required to counteract the global font-family !important rule applied to all * children.
