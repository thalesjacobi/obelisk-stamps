Instagram Carousel Generator

Project: Obelisk Stamps File: Technical Reference Date: February 2026

1. Overview

The Instagram Carousel Generator is an admin tool that transforms an article's content into a set of up to 10 square (1024×1024) images suitable for an Instagram carousel post. It uses a two-step AI pipeline:

  1. GPT-4.1-mini reads the article and writes N sequential scene descriptions — one DALL-E prompt and one short punchline (≤15 words) per scene.
  2. DALL-E 3 generates one image per prompt, with a consistent visual style suffix appended to every prompt.

Results stream to the browser in real time via Server-Sent Events (SSE).

Key capabilities

CapabilityDescription
Slot-based storage10 active slots (indices 0–9) + unlimited archived slots (10+). None in a slot means it's empty and eligible for generation.
Partial generationIf some slots are already filled, only the empty slots are generated — GPT is asked for exactly N scenes where N = empty slot count.
Per-image regenerationAny current image can be replaced by editing its prompt/punchline and clicking Re-run. The old image is archived, not deleted.
Archive AllMoves all current images to archived slots, clearing slots 0–9 for a fresh run.
PunchlinesEach image has a companion short caption used as an Instagram slide caption.
Style customisationA global "style suffix" is stored in site_settings and editable from the admin Settings tab. Each article also records the style used when its carousel was generated.
Real-time streamingImages appear in the UI one by one as they're generated, via SSE.

3. Database Schema

Table: articles — carousel columns

Four TEXT columns are added to the existing articles table to store carousel data.

ColumnTypePurpose
carousel_promptsTEXT (JSON array)DALL-E 3 prompts. Indices 0–9 = current; 10+ = archived.
carousel_imagesTEXT (JSON array)Relative file paths to generated images (articles/{id}/carousel/image_N.png). Same index convention.
carousel_punchlinesTEXT (JSON array)Short Instagram captions (≤15 words). Same index convention.
carousel_styleTEXTThe style suffix that was active when this article's carousel was last generated. Used by "View Style" modal.
Index convention: Slots 0–9 are the active/current carousel. Slots 10+ are archived (moved there by "Archive All" or per-image re-run). A null / None value in slots 0–9 means that slot is empty and eligible for generation.

Migration

The carousel columns are added at startup via an idempotent migration in init_articles_table(). Each ALTER TABLE ... ADD COLUMN is wrapped in a try/except so it silently skips if the column already exists: 

for col in ("carousel_prompts TEXT", "carousel_images TEXT",
            "carousel_punchlines TEXT", "carousel_style TEXT"):
    try:
        cur.execute(f"ALTER TABLE articles ADD COLUMN {col}")
    except Exception:
        pass  # column already exists

4. Python Backend (app.py)

Constant

CAROUSEL_STYLE_SUFFIX (line ~79) is the hardcoded default style string. It serves two purposes: it seeds the site_settings table on first run, and it acts as the fallback value if the DB is unavailable. 

CAROUSEL_STYLE_SUFFIX = (
    "Style: vintage editorial illustration, reminiscent of classic postage-stamp engravings "
    "and collector's print ephemera. Rich, deep colour palette — navy blue, burgundy, warm "
    "gold and ivory — with fine-line detail and a tactile, handcrafted quality. Strong focal "
    "composition with a sense of heritage and craftsmanship. "
    "Square 1:1 format for Instagram. No text, lettering, numbers, or watermarks in the image."
)

Helper Functions

get_setting(key, default=None)

Reads a single value from site_settings. Returns default if the key doesn't exist.

def get_setting(key, default=None):
    """Return a value from site_settings, or default if not found."""
    row = query_one("SELECT value FROM site_settings WHERE `key` = %s", (key,))
    return row[0] if row else default

Init Functions

init_articles_table()

Creates the articles table if it doesn't exist, then runs the idempotent carousel column migration shown in the Database section above. Called once at Flask startup.

init_site_settings()

Creates the site_settings table if it doesn't exist, then seeds the default carousel_style value on first run. INSERT IGNORE ensures the default is only written once — subsequent starts leave any admin-edited value untouched. 

def init_site_settings():
    conn = get_db()
    cur = conn.cursor()
    cur.execute("""
        CREATE TABLE IF NOT EXISTS site_settings (
            `key` VARCHAR(100) PRIMARY KEY,
            value TEXT,
            updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
        )
    """)
    cur.execute(
        "INSERT IGNORE INTO site_settings (`key`, value) VALUES ('carousel_style', %s)",
        (CAROUSEL_STYLE_SUFFIX,),
    )
    conn.commit()
    cur.close()
    conn.close()

API Routes

All routes require @login_required and @admin_required decorators.

MethodURLFunctionReturns
GET /admin/articles/<id>/carousel-images admin_article_carousel_images JSON: {images, prompts, punchlines}
POST /admin/articles/<id>/generate-carousel admin_article_generate_carousel SSE stream (text/event-stream)
POST /admin/articles/<id>/archive-carousel admin_article_archive_carousel JSON: {success, archived}
POST /admin/articles/<id>/regenerate-carousel-image admin_article_regenerate_carousel_image JSON: {url, prompt, punchline}
GET /admin/settings admin_settings Renders admin.html with active_tab="settings"
POST /admin/settings/save admin_settings_save JSON: {success: true} or {error: "..."}

Route detail: GET carousel-images

Reads carousel_prompts, carousel_images, carousel_punchlines from the articles row. Converts relative image paths to full static URLs via url_for("static", filename=...). Returns arrays for all three; null entries in the arrays indicate empty slots.

Route detail: POST generate-carousel

The main generation endpoint. Accepts no request body. Returns a streaming SSE response (see SSE Event Protocol below).

Key pre-flight logic (outside the streaming closure):

  • Reads active_style from site_settings.carousel_style (falls back to constant).
  • Loads existing carousel_images array, pads to 10 with None.
  • Computes empty_slots = list of 0-based indices where the image is None/null.
  • N = len(empty_slots). If N == 0, yields done immediately and returns.

Route detail: POST archive-carousel

Moves non-null items from slots 0–9 to the end of the arrays (after any existing archived items). Slots 0–9 are set to None. 

# After archiving, the arrays look like:
# [None, None, ..., None,   <- 10 empty current slots
#  <previously archived items>,
#  <newly archived items from slots 0-9>]

Returns {success: true, archived: N} where N is the count of items moved.

Route detail: POST regenerate-carousel-image

Request body: {"index": 0–9, "prompt": "...", "punchline": "..."}

  • Only indices 0–9 are valid (active slots).
  • New image saved as image_{index+1}_{timestamp}.png to preserve the old file.
  • Old prompt/image/punchline at that index are appended to the end of the arrays (archived).
  • The slot is updated in-place with the new values.
  • Also saves carousel_style = active_style to the article row.

Route detail: POST /admin/settings/save

Accepts {"carousel_style": "..."}. Uses MySQL upsert pattern: INSERT INTO ... ON DUPLICATE KEY UPDATE value = %s. Validates that the value is non-empty before writing.

5. SSE Event Protocol

The generate-carousel endpoint returns Content-Type: text/event-stream. Each event has the format: 

event: <type>
data: <JSON string>
Event typePayload fieldsWhen emitted
prompts {prompts: string[], punchlines: string[]} Immediately after GPT returns all N scene descriptions. Allows the UI to show "Generating images…" state.
image {index, seq, of, url, prompt, punchline} After each DALL-E image is downloaded and saved to disk. index = 1-based slot number; seq = position in current generation sequence; of = total images being generated (N).
error {index, message} When a single image fails (content policy, rate limit, etc.). Generation continues for remaining slots.
done {count} After all slots are processed. count = number of successfully generated images.
Why not EventSource? The browser's built-in EventSource API only supports GET requests. Since this endpoint uses POST, the frontend uses fetch() + ReadableStream to consume the SSE stream manually. See JS: SSE Streaming Pattern.

Progress calculation (frontend)

// In handleCarouselEvent(), on 'image' event:
const pct = p.of ? Math.round((p.seq / p.of) * 100) : (p.index * 10);
progressBar.style.width = pct + '%';
statusText.textContent = 'Generated ' + (p.seq || p.index) + ' of ' + (p.of || 10) + '…';

SSE Streaming Pattern

Because EventSource only supports GET, the frontend uses fetch() with an AbortController and reads the response body as a ReadableStream: 

carouselAbortCtrl = new AbortController();

fetch('/admin/articles/' + ARTICLE_ID + '/generate-carousel', {
    method: 'POST',
    signal: carouselAbortCtrl.signal
})
.then(res => {
    if (!res.ok) return res.json().then(d => { throw new Error(d.error || 'Server error'); });
    const reader = res.body.getReader();
    const dec = new TextDecoder();
    let buf = '';

    function read({ done, value }) {
        if (done) { carouselDone(-1); return; }
        buf += dec.decode(value, { stream: true });
        const parts = buf.split('\n\n');     // split on SSE event boundaries
        buf = parts.pop();                   // keep partial last chunk in buffer
        parts.forEach(part => {
            let ev = 'message', data = '';
            part.split('\n').forEach(line => {
                if (line.startsWith('event: ')) ev = line.slice(7).trim();
                if (line.startsWith('data: '))  data = line.slice(6).trim();
            });
            if (!data) return;
            try { handleCarouselEvent(ev, JSON.parse(data)); } catch(e) {}
        });
        reader.read().then(read);
    }
    reader.read().then(read);
})
.catch(err => {
    if (err.name === 'AbortError') { carouselStopped(); }
    else { /* show error, re-enable buttons */ }
});

8. End-to-End Logic Flows

A. Full Generation (all 10 slots empty)

  1. User clicks "Generate Carousel" button (or kebab menu item).
  2. startCarousel(): disables Generate buttons, shows Stop + progress bar. Adds placeholders cph-1 through cph-10 to the current grid.
  3. Frontend POSTs to /admin/articles/{id}/generate-carousel.
  4. Backend: reads active_style from site_settings. Loads existing arrays from DB (all null / empty → empty_slots = [0,1,...,9], N = 10).
  5. Backend: GPT-4.1-mini call — system prompt instructs it to generate exactly 10 {prompt, punchline} objects as a JSON array.
  6. Backend: yields event: prompts. Frontend updates status to "Generating images…".
  7. Backend: for each of the 10 slots — calls DALL-E 3 with "{prompt}. {active_style}"; downloads the image; saves to static/articles/{id}/carousel/image_{N}.png.
  8. Backend: yields event: image {index, seq, of=10, url, prompt, punchline}. Frontend: replaces placeholder with image tile, advances progress bar.
  9. After all slots: backend yields event: done {count}.
  10. Backend: UPDATE articles — writes carousel_prompts, carousel_punchlines, carousel_images, carousel_style.
  11. Frontend: carouselDone(count) — cleans up, re-enables buttons, calls updateCarouselTabCounts() which hides Generate button (all 10 slots now filled).

B. Partial Generation (some slots already filled)

  1. Same as Full Generation steps 1–3.
  2. Backend detects N < 10 empty slots. GPT is asked for exactly N scenes.
  3. Images are generated only for the N empty slot indices; existing images in filled slots are untouched.
  4. SSE image events: index = the actual slot number (1–10), seq = position in current generation (1–N), of = N.
  5. Frontend progress: (seq / of) × 100%. Status: "Generated {seq} of {N}…".
  6. DB save: merged_* arrays preserve existing filled slots; new values go into the correct empty indices.

C. Per-Image Re-run

  1. User clicks "i" button on a current image tile. openCarouselEditModal(index0, prompt, punchline, false) — modal opens in edit mode with pre-filled textareas.
  2. User edits prompt and/or punchline and clicks "Re-run this prompt".
  3. rerunCarouselImage() POSTs {index: 0–9, prompt, punchline} to regenerate-carousel-image.
  4. Backend validates index is 0–9. Reads existing arrays. Saves the old prompt/image/punchline at that index.
  5. Backend calls DALL-E 3. Saves new image as image_{index+1}_{timestamp}.png (timestamped to preserve old file for the Removed tab).
  6. Backend: updates the slot with new values; appends old values to the end of the arrays (slot 10+). Saves to DB.
  7. Frontend: addRemovedImage(oldTile) moves the old tile to the Removed grid. addCarouselImage() adds the new tile. Modal closes. updateCarouselTabCounts() runs.

D. Archive All

  1. User clicks "Archive All" button.
  2. archiveAllCarousel() POSTs to archive-carousel.
  3. Backend: collects non-null items from slots 0–9 as new_archived. Appends after any existing archived items. Resets slots 0–9 to None. Saves.
  4. Frontend: all cimg-* tiles are moved to removed grid via addRemovedImage(). Current grid is now empty. updateCarouselTabCounts(): Generate button becomes visible.

E. Style Change

  1. Admin navigates to /admin/settings. Settings tab shows the current site_settings.carousel_style in a textarea.
  2. Admin edits text and clicks "Save Settings".
  3. saveSettings() POSTs {carousel_style: "..."} to /admin/settings/save.
  4. Backend: upserts via INSERT ... ON DUPLICATE KEY UPDATE. Returns {success: true}.
  5. From the next carousel generation onwards, the new style is used. The article's carousel_style column is updated to record which style produced its images.
  6. Existing articles' "View Style" modals show their own saved carousel_style value, not the new site setting.

9. Slot Index Convention

All three carousel arrays (carousel_prompts, carousel_images, carousel_punchlines) share the same index convention:

Index rangeMeaningUI location
0–9Active / current carousel slots"Current" tab in carousel editor
10+Archived / removed items"Removed" tab in carousel editor

A null / None entry in slots 0–9 means that slot is empty. The Generate button and kebab menu item are enabled only when 10 - currentCount > 0, i.e., at least one slot is empty.

Array padding: Before computing empty_slots, the existing arrays are padded to exactly 10 entries: (existing[:10] + [None] * 10)[:10]. This ensures empty_slots is always computed against a 10-element list, even if the DB array is shorter (e.g., a partial generation that stopped at 5).

Archive boundary preservation

The archive-carousel route writes a [None] * 10 prefix before the archived items. This ensures that existing[10:] in generate-carousel always correctly slices only truly archived items, regardless of how many were archived: 

# After archive-carousel, arrays always look like:
final_p  = [None] * 10 + already_archived_p  + new_archived_p
final_i  = [None] * 10 + already_archived_i  + new_archived_i
final_pl = [None] * 10 + already_archived_pl + new_archived_pl
# So existing[10:] == already_archived + new_archived  ✓
