Carousel Generator — Technical Documentation

Source: app.py, templates/article_edit.html Last updated: 2026-03-22

1. Overview

The carousel generator creates up to 10 square carousel images via DALL-E 3 from article content. The system breaks article text into scenes using GPT-4.1-mini, generates a DALL-E image for each scene, composites punchline text overlays via Pillow, and then enables posting the finished carousel to 6 social media platforms: Instagram, Facebook, X (Twitter), Threads, Pinterest, and LinkedIn.

AttributeValue
Max slides10 (active slots indexed 0–9; archived images stored at index 10+)
Generation time~60–90 seconds for a full 10-slide carousel
Cost per run~$0.40 ($0.04 per DALL-E image)
DALL-E native size1024×1024
Composed output size960×960 (with punchline text overlays)
Storyboard modelGPT-4.1-mini (max_tokens 3000)
Caption modelGPT-4.1-mini (max_tokens 600)
Supported platformsInstagram, Facebook, X, Threads, Pinterest, LinkedIn
Images are generated at 1024×1024 by DALL-E 3, then composed down to 960×960 with punchline text overlays before posting to social platforms. Each platform receives its own composed copy (with platform-specific hint text where applicable).

2. Database Schema

Articles Table Columns

The articles table includes these carousel-specific columns:

ColumnTypeDescription

carousel_styleTEXTThe DALL-E style suffix that was used for the most recent generation run.

The active/archived split uses a single JSON array with a positional convention: indices 0–9 are “live” slots and indices 10+ are archived. The carousel_archived_meta dict provides provenance for each archived entry.

3. Image Generation Pipeline

GPT-4.1-mini Storyboard Call

Before generating any images, the system breaks the article into N scenes (up to 10) using a GPT call:

ParameterValue
Modelgpt-4.1-mini
max_tokens3000
InputArticle title + body text
OutputJSON array of objects, each with a prompt (DALL-E scene description) and punchline (max 15 words, short caption for overlay)

The storyboard call instructs GPT to decompose the article into visually distinct scenes, writing a detailed DALL-E prompt for each scene and a concise punchline (max 15 words) that summarizes the scene's key point for text overlay.

DALL-E 3 Image Generation

Each scene prompt is sent to DALL-E 3 individually:

ParameterValue
Modeldall-e-3
Size1024x1024
Qualitystandard
Stylevivid
n1

Images are generated one at a time (n=1) in sequence. After each image completes, the result is immediately saved to the database so partial results survive interruptions. The generation route uses SSE (Server-Sent Events) to stream progress to the frontend, emitting prompts, image, error, and done events.

Style Suffix

Every DALL-E prompt has a style suffix appended to it before submission. The resolution order is:

  1. Value from site_settings key carousel_style (editable at /admin/settings)
  2. If not set, falls back to the hardcoded constant CAROUSEL_STYLE_SUFFIX in app.py

The style suffix ensures visual consistency across all slides (e.g., specifying art style, color palette, composition rules).

Storage

Each generated image is stored in two locations:

  • Local: static/articles/{id}/carousel/image_{slot}.png
  • GCS: articles/{id}/carousel/image_{slot}.png (via upload_bytes_to_gcs())

The GCS URL is stored in the carousel_images JSON array. If GCS is not configured, the local static path is used as a fallback.

Generation costs ~$0.04 per image ($0.40 for a full 10-slide carousel). The worker runs in a daemon thread and updates the DB after each individual slide completes, so partial results are preserved if the process is interrupted.

4. Image Compositing

Two key functions in app.py handle compositing punchline text overlays onto generated images using Pillow.

compose_carousel_slide()

compose_carousel_slide(img_bytes, punchline, slide_index, total_slides, width=960, height=960, band_top=None, hint_text=None)

Overlay Elements

  • Dark gradient/band: semi-transparent dark band at the bottom (~92% opacity) with a gradient fade above it
  • Punchline text: white uppercase text, word-wrapped, left-aligned, using Roboto-Bold font
  • Swipe hint: gold-colored text “Slide right for more >>>” on all slides except the last
  • Facebook override: when hint_text is passed (e.g., "obelisk-stamps.com"), it replaces the default swipe hint on all slides including the last

Technical Details

  • Uses Pillow (PIL) for all rendering
  • Font: Roboto-Bold (loaded from embedded font file at _FONT_PATH)
  • Font sizes: 4.8% of width for main punchline text, 2.1% of width for the slide hint
  • Output format: JPEG, quality=92
  • The band_top parameter controls the Y position where the dark overlay band begins. When None, it is computed per-slide. Pass a pre-computed value for consistent band placement across all slides.

_compute_max_overlay_band_top()

_compute_max_overlay_band_top(punchlines, width=960, height=960)
  • Pre-computes a consistent Y position for the text band across all slides
  • Iterates all punchlines, calculates the required band height for each one
  • Returns the minimum band_top value (i.e., the tallest band needed)
  • Ensures all slides have identical band placement regardless of text length differences
Always call _compute_max_overlay_band_top() before composing a batch of slides. Pass the returned value as band_top to each compose_carousel_slide() call to guarantee uniform visual appearance.

5. Caption Generation

POST /admin/articles/<id>/generate-ig-caption

ParameterValue
Modelgpt-4.1-mini
max_tokens600
Body{component: "carousel"|"cinemagraph"|"narrated"}

Prompt Resolution Order

  1. Per-article override: if the article has a custom caption prompt stored, use that
  2. Global setting: if a global caption prompt is configured in site_settings, use that
  3. Hardcoded default: falls back to _IG_CAPTION_DEFAULT_PROMPT constant in app.py

Post-Processing

  • If SITE_URL environment variable is set, a call-to-action with the article link is appended to the generated caption
  • Returns: {caption: "..."}
  • Cost: ~$0.01 per generation

7. Social Media Posting — Platform Details

Instagram

API: Facebook Graph API v21.0  |  Worker: _post_to_instagram_worker(article_id, caption, post_type="car")

Posting Flow

  1. Load carousel images and punchlines from DB
  2. Compute consistent band_top via _compute_max_overlay_band_top()
  3. For each slide: compose overlay via compose_carousel_slide(), upload composed JPEG to GCS
  4. Create child containers: POST /{ig_user_id}/media with is_carousel_item=true, image_url pointing to GCS
  5. Poll each child container until status is FINISHED (5-minute timeout, exponential backoff)
  6. Create carousel container: POST /{ig_user_id}/media with media_type=CAROUSEL, children array, caption
  7. Poll carousel container until FINISHED
  8. Publish: POST /{ig_user_id}/media_publish with creation_id
  9. Fetch permalink from the published media
  10. Store media_id, permalink, and caption snapshot in site_settings
Rate limit handling: The _ig_api_call() helper auto-retries once after a 30-second pause if the Graph API returns a rate-limit error.

Facebook

API: Facebook Graph API v21.0  |  Worker: _post_to_facebook_worker(article_id, caption, post_type="car")

Posting Flow

  1. Load carousel images and punchlines from DB
  2. Compute consistent band_top
  3. For each slide: compose overlay with hint_text="obelisk-stamps.com" (instead of the default swipe hint)
  4. Upload each composed image as an unpublished photo: POST /{page_id}/photos with published=false
  5. Create the feed post: POST /{page_id}/feed with message=caption and attached_media array referencing the unpublished photo IDs
  6. Store post_id, permalink, and caption snapshot
Facebook carousel posts use the unpublished photo + attached_media pattern. The composed images show "obelisk-stamps.com" as the hint text on every slide instead of the default swipe hint.

X (Twitter)

Auth: OAuth 1.0a via requests_oauthlib  |  Worker: _post_carousel_x_worker(article_id, caption, component)

Posting Flow

  1. Compose each slide with punchline overlay
  2. Upload each image: POST upload.twitter.com/1.1/media/upload.json
  3. If more than 4 images: split into groups of 4 per tweet (X limit is 4 images per tweet)
  4. Create first tweet with caption + first batch of media_ids: POST api.twitter.com/2/tweets
  5. Create reply tweets for remaining batches, numbered “2/3”, “3/3” etc., each replying to the previous tweet
  6. Store tweet_id of the first tweet
X limits media attachments to 4 images per tweet. A 10-slide carousel becomes a thread of 3 tweets (4 + 4 + 2 images). The first tweet carries the full caption; subsequent tweets are numbered replies.

Threads

API: graph.threads.net/v1.0  |  Worker: _post_carousel_threads_worker(article_id, caption, component)

Posting Flow

  1. Compose each slide with punchline overlay
  2. Upload composed images to GCS for public URL access
  3. Create child containers: POST /{user_id}/threads with media_type=IMAGE, is_carousel_item=true, image_url
  4. Create carousel container: POST /{user_id}/threads with media_type=CAROUSEL, children array, text=caption[:500]
  5. Poll carousel container until status is FINISHED (5-minute timeout)
  6. Publish: POST /{user_id}/threads_publish with creation_id
  7. Fetch permalink from published thread
  8. Store post_id, permalink, and caption snapshot
Threads caption limit is 500 characters. Captions are automatically truncated: caption[:500].

Pinterest

API: api.pinterest.com/v5  |  Auth: Bearer token  |  Worker: _post_carousel_pinterest_worker(article_id, caption, component)

Posting Flow

  1. Compose each slide with punchline overlay
  2. Upload composed images to GCS for public URL access
  3. Split images into chunks of 5 (Pinterest carousel supports 2–5 images per pin)
  4. For chunks with 2+ images: create pin via POST /v5/pins with media_source.source_type=multiple_image_urls and media_source.items array
  5. For single remaining images: create pin with media_source.source_type=image_url
  6. Each pin includes: board_id from env, title=caption[:100], description=caption[:800], link to article URL
  7. Store pin_id(s) and permalink(s)
Pinterest carousel pins hold 2–5 images. A 10-slide carousel becomes 2 pins (5 + 5). Title is truncated to 100 characters and description to 800 characters.

LinkedIn

API: api.linkedin.com/rest  |  Auth: Bearer token  |  Worker: _post_carousel_linkedin_worker(article_id, caption, component)

Required Headers

Authorization: Bearer {LINKEDIN_ACCESS_TOKEN}
LinkedIn-Version: 202502
X-Restli-Protocol-Version: 2.0.0

Author URN

urn:li:organization:{LINKEDIN_ORG_ID}

Posting Flow

  1. Compose each slide with punchline overlay
  2. Upload composed images to GCS for public URL access
  3. For each image: initialize upload via POST /images?action=initializeUpload (returns an upload URL and image URN)
  4. Upload image bytes via PUT to the returned upload URL
  5. Create post via POST /posts:
    • For multiple images: use multiImage.images array with each image's URN
    • For single image: use media.id with the image URN
  6. Post body includes: author (org URN), commentary=caption[:3000], visibility=PUBLIC, distribution.feedDistribution=MAIN_FEED
  7. Store post URN, permalink, and caption snapshot
LinkedIn caption limit is 3000 characters. The post is published under the organization identity (not a personal profile). The LinkedIn-Version: 202502 header is required for all API calls.

8. Archive & Delete Flow

Archive Carousel Images

  • Moves all images from active slots (0–9) to archived positions (10+) in the carousel_images, carousel_prompts, and carousel_punchlines JSON arrays
  • Populates carousel_archived_meta with an entry for each archived image containing:
    • archived_at: ISO 8601 timestamp of when the archive happened
    • slot_source: the original active slot index (0–9) the image occupied
  • Clears the active slots (sets indices 0–9 to null)

Restore Archived Image

  • Accepts an archived index (10+) and a target slot (0–9)
  • If the target slot already contains an image, that image is automatically archived first (displaced to 10+)
  • The archived image is moved to the target active slot, and its archived_meta entry is removed

Archive Social Post

  • Takes the current live post data (media_id/post_id, permalink, caption snapshot) and appends it to the platform's history JSON array in site_settings
  • Clears the active post keys (status, result, media_id, snapshot) for that platform
  • Does not delete the post from the social platform itself

Delete Social Post (FB, X, Pinterest, LinkedIn)

  • Calls the platform's DELETE API to remove the post from the social network:
    • Facebook: DELETE /{post_id} via Graph API
    • X: DELETE /2/tweets/{tweet_id}
    • Pinterest: DELETE /v5/pins/{pin_id}
    • LinkedIn: DELETE /posts/{post_urn}
  • After successful deletion, archives the post locally (same as archive flow)
  • Instagram does not support deletion via the Graph API, so no delete route exists for IG

10. Past Posts History

Archived (past) posts are stored as JSON arrays in site_settings, using the key pattern:

{platform}_{component}_history_{article_id}

For example: ig_car_post_history_42, x_carousel_history_42, linkedin_carousel_history_42.

Each entry in the history array contains:

Polling & SSE Patterns

SSE for Image Generation

The POST /admin/articles/<id>/generate-carousel route returns an SSE stream. The frontend opens an EventSource connection and listens for these event types:

EventDataFrontend Action
promptsJSON array of storyboard prompts + punchlinesDisplays prompts in the UI, prepares grid slots
imageJSON with index, url, promptAdds the generated image to the grid at the specified index
errorError message stringShows error alert
doneTotal count of generated imagesFinalizes generation UI, enables post buttons

Status Polling for Social Posting

All social media posting follows the same async pattern:

  1. POST to the platform's posting route — returns immediately with {ok: true}
  2. Frontend starts a setInterval polling the platform's status endpoint every 5 seconds
  3. Status transitions: null → running:step description → done:permalink or error:message
  4. On done: — calls _makeViewPostBtn() to transform the post button to a “view post” link
  5. On error: — shows a dismissible Bootstrap alert with the error message
  6. Polling interval is cleared once a terminal state (done: or error:) is received

Caption Auto-Population

On DOMContentLoaded

  • Fills empty caption textareas for Threads, Pinterest, LinkedIn, and X with defaultCaption (article title + article URL)
  • Instagram caption is NOT auto-filled — it has a dedicated “Generate caption” link that calls GPT

Copy from Instagram

  • Each non-Instagram platform section has a “Copy from Instagram” link
  • Copies the IG caption textarea value to the target platform's textarea
  • Truncates to the target platform's character limit (e.g., 500 for Threads, 3000 for LinkedIn)

Icon System

  • Section headers: use inline SVGs (Instagram camera, Facebook F, X logo) or Font Awesome brand icons (Threads, Pinterest, LinkedIn)
  • Post buttons in body containers: use FA brand icons: <i class="fa-brands fa-{platform}">
  • “Already posted” buttons: use the _platformIconSvg map (FA brand icons) combined with _extLinkSvg (inline SVG external link icon)
CSS fix: font-family: "Font Awesome 6 Brands" !important override is applied to .fa-brands elements inside social body containers. This is required to counteract the global font-family !important rule applied to all * children within those containers.
